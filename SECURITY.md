# Security Policy

## Supported Versions

This project is only deployed from `master`. All security fixes are applied to the current code on `master`.

Versions prior to 1.0 are not supported. From 1.0 onwards, only the latest released version is supported.

## Reporting a Vulnerability

Please do NOT open a public issue for security vulnerabilities.

Instead, use GitHub's private vulnerability reporting (Security tab → Report a vulnerability).

Include: description, reproduction steps, potential impact, affected component.

## Response Time

I will respond to vulnerability reports as soon as possible.

Acknowledgment usually within a few business days, depending on severity and my schedule.

## Scope

In scope: application code

Out of scope: third-party dependencies (report upstream), social engineering, physical access.
